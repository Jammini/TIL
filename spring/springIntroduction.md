# 객체 지향 설계와 스프링

## 1. 스프링이란?

[spring.io](http://spring.io) 를 보면 여러가지가 존재하는 것을 볼 수 있다.

이 모든 것이 스프링 프레임워크이고 이것을 편리하게 도와주는 것이 스프링 부트.

![image](https://user-images.githubusercontent.com/59176149/212706619-652e9562-6dbe-44fb-a3db-fe2b8d854b4d.png)


## 2. 스프링 부트

- 스프링을 편리하게 사용할 수 있도록 지원, 최근에는 기본으로 사용
- 단독으로 실행할 수 있는 스프링 애플리케이션을 쉽게 생성
- Tomcat 같은 웹 서버를 내장해도 별도의 웹 서버를 설치하지 않아도 됨.
- 손쉬운 빌드 구성을 위한 starter 종속성 제공
- 스프링과 3rd parth(외부) 라이브러리 자동 구성
- 메트릭, 상태확인, 외부 구성 같은 프로덕션 준비 기능 제공
- 관례에 의한 간결한 설정

## 3 . 스프링 단어?

- 스프링이라는 단어는 문맥에 따라 다르게 사용된다.
    - 스프링 DI 컨테이너 기술
    - 스프링 프레임워크
    - 스프링 부트, 스프링 프레임워크 등을 모두 포함한 스프링 생태계

## 4. 스프링의 진짜 핵심

- 스프링은 자바 언어 기반의 프레임워크
- 자바 언어의 가장 큰 특징 - 객체 지향 언어
- 스프링은 객체 지향 언어가 가진 강혁한 특징을 살려내는 프레임워크
- 스프링은 좋은 객체 지향 애플리케이션을 개발할 수 있게 도와주는 프레임워크
